# 📋 Система Логирования Ошибок

> Структурированное отслеживание всех ошибок для анализа и улучшения системы

## 🗂️ Структура папок

```
logs/
├── development/        # Ошибки во время разработки (Claude Code)
├── agents/             # Ошибки агентов в работе
│   ├── listener/       # Ошибки Агента-Слушателя
│   ├── manager/        # Ошибки Агента-Менеджера
│   ├── vision/         # Ошибки Агента-Vision
│   ├── engineer/       # Ошибки Агента-Инженера
│   ├── generator/      # Ошибки Агента-Генератора
│   └── delivery/       # Ошибки Агента-Доставщика
├── errors/             # Критические системные ошибки
└── system/             # Системные логи (State Machine, Database)
```

---

## 📝 Типы логов

### 1. **development/** - Ошибки разработки
**Когда записывать:**
- Ошибки во время написания кода
- Проблемы с настройкой
- Тестовые ошибки
- Баги при отладке

**Формат файла:** `YYYY-MM-DD_HH-MM-SS_error.json`

**Пример:** `2024-01-15_14-30-45_database_connection.json`

---

### 2. **agents/[agent_name]/** - Ошибки агентов
**Когда записывать:**
- Агент не смог обработать запрос
- Превышен лимит токенов
- Неверный ответ от API
- Ошибка при генерации промпта

**Формат файла:** `YYYY-MM-DD_order-ID_error-type.json`

**Пример:** `2024-01-15_order-42_token_limit.json`

---

### 3. **errors/** - Критические ошибки
**Когда записывать:**
- Система полностью упала
- База данных недоступна
- Критическая ошибка безопасности
- Потеря данных

**Формат файла:** `YYYY-MM-DD_HH-MM-SS_CRITICAL.json`

---

### 4. **system/** - Системные логи
**Когда записывать:**
- Переходы состояний (State Machine)
- Подключения/отключения от сервисов
- Запуск/остановка бота
- Изменения конфигурации

**Формат файла:** `YYYY-MM-DD_system.log`

## 🎯 Уровни серьезности (severity)

| Уровень | Описание | Действие |
|---------|----------|----------|
| **critical** | Система не работает | Немедленное вмешательство |
| **high** | Заказ заблокирован | Исправить в течение часа |
| **medium** | Агент ошибся, но система работает | Исправить в течение дня |
| **low** | Минорная проблема | Можно отложить |
| **info** | Информационное сообщение | Для статистики |

## 📊 Анализ ошибок

### Еженедельный анализ:
```bash
# Посмотреть все ошибки за неделю
find logs/ -name "*.json" -mtime -7

# Подсчитать ошибки по агентам
grep -r "agent_name" logs/agents/ | sort | uniq -c

# Найти критические ошибки
grep -r "\"severity\": \"critical\"" logs/
```

## 🔧 Автоматическое логирование

Система автоматически создает отчеты при ошибках через:
- `src/utils/logger.py` - основной логгер
- `src/utils/error_reporter.py` - создание отчетов

### Пример использования в коде:

```python
from src.utils.error_reporter import ErrorReporter

reporter = ErrorReporter()

try:
    # Код агента
    result = agent.process()
except Exception as e:
    # Автоматически создает отчет
    reporter.log_error(
        agent_name="manager",
        order_id=order_id,
        error=e,
        context={"state": current_state}
    )
```

## 📈 Метрики для отслеживания

1. **Частота ошибок** - сколько ошибок в день/неделю
2. **Время до решения** - как быстро исправляются
3. **Повторяющиеся ошибки** - что происходит регулярно
4. **Затронутые пользователи** - сколько клиентов пострадало
5. **Стоимость ошибок** - потерянные токены/деньги

## 🗑️ Очистка старых логов

```bash
# Удалить логи старше 30 дней
find logs/ -name "*.json" -mtime +30 -delete

# Архивировать логи старше 7 дней
find logs/ -name "*.json" -mtime +7 -exec gzip {} \;
```
