# Telegram Бот для Приёма Заказов на AI-Изображения

Простой MVP-бот для автоматизации приёма заказов через Telegram.
Клиент описывает что хочет, платит — ты получаешь промпт от Claude, генеришь картинку и отправляешь.

## Как это работает

```
Клиент: /start → описание → стиль → скрин оплаты
                                          ↓
Ты (admin): видишь уведомление → [Подтвердить] → получаешь промпт от Claude
                                                          ↓
                              генеришь картинку → [Доставить] → отправляешь фото
                                                                      ↓
                                                             Клиент получает результат
```

## Быстрый старт

### 1. Установить зависимости

```bash
pip install -r requirements.txt
```

### 2. Создать .env файл

```bash
cp .env.example .env
```

Открой `.env` и заполни 5 переменных:
- `SUPABASE_URL` и `SUPABASE_KEY` — из Supabase Dashboard → Settings → API
- `TELEGRAM_BOT_TOKEN` — от @BotFather
- `TELEGRAM_ADMIN_IDS` — твой Telegram ID (узнать у @userinfobot)
- `ANTHROPIC_API_KEY` — из console.anthropic.com

### 3. Создать таблицы в Supabase

```bash
python scripts/setup_database.py
```

Скопируй выведенный SQL → вставь в Supabase Dashboard → SQL Editor → Run.

### 4. Прописать свои реквизиты для оплаты

Открой [src/integrations/telegram/bot.py](src/integrations/telegram/bot.py) и найди строки:
```python
f"Сбербанк: 1234 5678 9012 3456\n"
f"Получатель: Иванов И.И.\n"
```
Замени на свои реквизиты.

### 5. Запустить

```bash
python main.py
```

## Структура (только важные файлы)

```
main.py                              ← точка входа
config/settings.py                   ← настройки из .env
src/
  core/database.py                   ← работа с Supabase
  integrations/telegram/bot.py       ← весь бот
scripts/
  setup_database.py                  ← SQL для создания таблиц
  run_bot.py                         ← альтернативный запуск
```

## Команды и кнопки

**Для клиентов:**
- `/start` — начать новый заказ
- `/cancel` — отменить текущий заказ

**Для тебя (admin):**
- `/orders` — показать последние 10 заказов (если пропустил уведомление)
- Кнопка **Подтвердить** — подтверждает оплату и запускает генерацию промпта
- Кнопка **Отклонить** — отклоняет оплату, уведомляет клиента
- Кнопка **Доставить клиенту** — переходит в режим доставки (следующим сообщением отправь фото)
